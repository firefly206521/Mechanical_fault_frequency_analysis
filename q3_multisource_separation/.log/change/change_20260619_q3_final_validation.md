# Q3 第三阶段修改日志与近频改进建议

## 范围

本次修改围绕第三问多故障源分离与定位代码完成，目标是：

1. 修复审查报告指出的运行时间统计、近频构造偏置等问题。
2. 加入并行 Monte Carlo，降低 200 次级别验证的等待时间。
3. 补全极端工况测试，形成阶段性最终检验。
4. 保留 Q3 主算法不依赖固定中心频率，避免偏离真实多源故障场景。

## 已完成修改

### 1. 运行时间统计修复

- 修复 `estimated_full_seconds` 使用固定默认参数估算的问题。
- 当前估算已基于命令行实际传入的 `--simulation-runs`、`--resolution-runs`、`--null-runs`、`--extreme-runs`。
- 并行后不再简单累加每个 trial 的 CPU 耗时作为墙钟时间估计，改用阶段墙钟耗时记录。
- 输出文件 `q3_runtime.txt` 中记录：
  - 目标运行次数；
  - 实际完成行数；
  - 各阶段耗时；
  - 总耗时；
  - worker 数量；
  - 经验误报率；
  - 等幅和不等幅近频辨识极限。

### 2. 近频构造偏置修复

- 移除了 `close_pair_resolver` 中围绕 `13.5 Hz` 强制左右对称构造的约束。
- 当前近频求解只要求频率有序，不再要求固定中心频率。
- 真实多源数据检测仍采用多峰 GLRT + BIC + 联合谐波回归，不假设故障频率围绕某个中心分布。
- 固定中心或对称间隔只保留为受控分辨率实验的设计方式，用于研究“频率间隔 delta 对识别率的影响”，不进入真实数据主流程。

### 3. 计算性能优化

- 增加 `--workers` 参数，默认使用 `min(8, os.cpu_count())`。
- Monte Carlo 仿真、近频分辨率实验、零假设误报实验、极端工况实验均支持多进程并行。
- GLRT 阈值由主进程预计算后传给 worker，避免每个 trial 重复做 Monte Carlo 阈值标定。
- 在追加 CSV 后直接复用内存中的结果行，避免每个阶段完成后反复读取 CSV。

### 4. 极端工况测试补全

新增极端工况验证集：

- `very_low_snr_-25`：极低信噪比。
- `very_low_snr_-30`：更极端低信噪比。
- `non_symmetric_close`：非对称近频。
- `identical_frequency`：完全同频，不具备单通道可辨识性。
- `phase_cancellation`：近似相消，不具备稳定单通道可辨识性。
- `overcomplete_k8`：更多故障源。
- `impulsive_noise`：脉冲噪声。
- `low_frequency_boundary`：低频边界。
- `high_frequency_boundary`：高频边界。

- `q3_extreme_summary.csv`

## 主要验证结论

### 1. 常规多源仿真

| SNR dB | 正确源数识别率 |
| ---: | ---: |
| -20 | 0.015 |
| -15 | 0.805 |
| -12 | 1.000 |
| -10 | 1.000 |
| -5 | 1.000 |

结论：当前方法在 `-12 dB` 及以上已稳定识别，在 `-15 dB` 附近进入过渡区，在 `-20 dB` 下检出能力明显不足。

### 2. 近频分辨率

- 等幅近频经验辨识极限约为 `0.0025 Hz`。
- 不等幅近频经验辨识极限约为 `0.003 Hz`。
- 不等幅近频比等幅近频更难，弱分量容易被强分量旁瓣、泄漏和联合拟合偏差吞没。

### 3. 极端工况

| 工况 | 成功率 | 说明 |
| --- | ---: | --- |
| very_low_snr_-25 | 0.000 | 极低 SNR 下算法无法稳定识别 |
| very_low_snr_-30 | 0.000 | 检测统计量基本被噪声主导 |
| non_symmetric_close | 0.730 | 非对称近频仍可识别，但低于常规间隔 |

| high_frequency_boundary | 1.000 | 高频边界表现稳定 |

## 当前剩余风险

1. 非对称近频成功率为 `0.73`，还不是论文主结论中可以宣称“稳定识别”的水平。
2. 完全同频和强相消不是代码 bug，而是单通道模型的理论不可辨识边界，需要在论文中明确说明。
3. `overcomplete_k8` 虽然成功率高，但平均运行时间显著增加，后续如果要大量扫描更多源数，需要继续优化。
4. 当前主要验证的是正弦叠加模型；若真实机械故障呈现冲击调制或非平稳调制，需要增加包络谱、循环平稳或时频方法作为补充。

## 提高非对称近频成功率的建议

### 建议 1：采用“先粗检、再局部联合细化”的两阶段近频流程

当前近频难点主要不是粗略发现频带，而是两个相邻分量在联合拟合时互相干扰。建议流程：

1. 用多峰 GLRT 找到疑似频带。
2. 对每个疑似近频簇建立局部候选频率集合。
3. 在局部簇内做二源或三源联合非线性最小二乘。
4. 用 BIC 或残差下降量判断是否真的需要拆成多个频率。

优点：比全局盲目增加源数更稳，也不会把真实单峰过度拆分。

### 建议 2：对强弱不等幅近频加入逐次剥离与残差复检

不等幅近频失败常见原因是强分量被检出后，弱分量被强分量谱泄漏掩盖。可以增加：

1. 先拟合最强分量。
2. 从原信号中减去该分量。
3. 对残差信号重新运行局部 GLRT。
4. 若残差中仍有显著峰，再和原强分量做联合重拟合。

关键点：最终结果必须回到联合拟合，不能只依赖逐次剥离，否则早期估计误差会传递。

### 建议 3：近频簇内使用更细的频率网格与自适应搜索窗口

非对称近频的弱分量可能落在固定网格的偏斜位置。建议：

- 对疑似近频区域使用比全局网格更密的局部网格；
- 局部搜索宽度根据频谱峰宽和当前估计间隔自适应设置；
- 对候选峰做抛物线插值或一维最小二乘精修后再进入联合拟合。

这类优化实现成本较低，通常能直接降低频率栅格误差。

### 建议 4：在模型选择中增加“近频惩罚修正”

标准 BIC 对近频弱分量可能偏保守，因为新增频率参数带来的残差下降不一定足够明显。可以在近频簇中尝试：

- 对候选二源模型和单源模型做局部似然比检验；
- 对近频簇单独降低拆分阈值；
- 只在局部残差存在结构性周期峰时放宽惩罚。

这能提升近频召回率，但会增加误拆风险，需要配合零假设仿真控制误报。

### 建议 5：论文表达上区分“经验辨识极限”和“理论不可辨识”

建议把结果写成：

- 当两个频率不同且间隔大于约 `0.003 Hz` 时，不等幅近频可较稳定辨识。
- 当间隔继续减小，强弱分量会出现合并、漏检或频率偏移。
- 当两个源完全同频或几乎相消时，单通道观测不具备唯一分解条件，需要多传感器或先验约束，这正好引出第四问的传感器布局意义。

这样不会把极端失败包装成算法缺陷，也能自然连接到 Q4 的鲁棒性优化。

## 推荐下一步

优先级从高到低：

1. 实现“强分量剥离 + 残差复检 + 联合重拟合”，目标提升 `non_symmetric_close` 成功率。
2. 增加局部近频细网格搜索，目标降低非对称频率偏移。
3. 重跑 `non_symmetric_close` 小型对比实验，例如每版 100 次，比较成功率和误报率。
4. 若成功率明显提升，再跑一次完整大测试，防止近频增强带来误报上升。
