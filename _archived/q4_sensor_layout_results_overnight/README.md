# Q4 V0 Overnight Results

This directory is the authoritative long-run validation set for Q4 V0.

Q4 V0 optimizes sensor placement over a finite set of candidate installable regions. It does not claim a continuous physical-space global optimum for an unknown machine.

## Directory Structure

- `pc1/`: overnight jobs from PC1.
- `pc2/`: overnight jobs from PC2.
- `_summary/`: merged summary CSVs generated by `q4_sensor_layout.summarize_overnight`.
- `pc1_run_log.txt`, `pc2_run_log.txt`: command, start/end time, and return code logs.

Each job directory contains:

- `paper/`: report, summary CSVs, and figures suitable for paper-facing use.
- `raw/`: trial rows, false-alarm rows, runtime parameters, random seeds, and complete trace data.

## Authoritative Summary Files

- `_summary/q4_overnight_job_summary.csv`
- `_summary/q4_overnight_top_layouts.csv`
- `_summary/q4_overnight_benchmark_snr.csv`

## V0 Conclusion

- Main recommended finite-candidate layout: `bearing_left+gearbox_left+output_shaft`.
- Close robustness alternative: `bearing_left+bearing_right+gearbox_left`.
- The main layout won both standard 40 s baseline seeds and the strict false-alarm profile.
- The alternative won low-SNR dense, long-window, and score-sensitivity profiles.

For paper wording, describe these as candidate regions under the assumed sensitivity/noise model, not as real-machine coordinates.
