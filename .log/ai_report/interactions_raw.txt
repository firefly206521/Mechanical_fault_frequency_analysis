# Q1 多模型实现与比较计划

## Summary
- 只解决 Q1：从 `data.xlsx` 的“单源故障”sheet 中检测微弱周期信号，并估计目标频率 `f0`。
- 实现多个模型，统一输入、统一输出、统一评价指标，先比较优劣，再决定 Q2/Q3 后续主线。
- 第一版建议实现 6 个模型：传统 FFT、Welch PSD、匹配滤波/GLRT、CFAR-FFT、SSA 去噪 + FFT、MUSIC。它们覆盖基线、统计判决、去噪增强、高分辨率估计。

## Models
- `FFT Peak Baseline`
  - 去均值、去趋势、Hann 窗，直接取频谱最大峰。
  - 用作传统傅里叶基线，便于说明改进幅度。
- `Welch PSD`
  - 分段平均功率谱，降低随机噪声方差。
  - 对比单次 FFT 的稳定性，但频率分辨率会下降。
- `CFAR + FFT`
  - 用滑动中位数估计局部噪声底，对频谱峰做自适应门限。
  - 输出 `f0_hat`、峰值/局部背景比、是否过门限。
- `Matched Filter / GLRT`
  - 在候选频率网格上计算正弦/余弦投影能量。
  - 用纯噪声 Monte Carlo 估计阈值，控制 `P_FA=0.05`。
  - 作为 Q1 主推荐候选，因为它和题面正弦模型最一致。
- `SSA Denoise + FFT`
  - 构造 Hankel 轨迹矩阵，保留前一对主要奇异分量重构，再做 FFT。
  - 用于验证去噪增强是否提升弱周期检测。
- `MUSIC`
  - 用延迟嵌入协方差矩阵和噪声子空间估计频率。
  - 作为高分辨率对照，重点看是否比 FFT 网格更精细、是否在低 SNR 下稳定。

# Q4 传感器布局与鲁棒性优化实施计划

## Summary

第四问新增一个独立模块 `q4_sensor_layout/`，不改 Q1-Q3 的核心算法。它把前三问的检测与分离能力包装成“多传感器融合检测器”，再用候选测点枚举和 Monte Carlo 仿真选择最多 3 个传感器布局。由于题目没有给真实几何结构，布局优化采用“候选测点 + 敏感度矩阵 + 噪声场景集”的鲁棒建模方式。

## Key Changes

- 建立多传感器观测模型：
  \[
  y_m(t)=\sum_k h_{mk}A_k\sin(2\pi f_kt+\phi_k+\delta_{mk})+n_m(t)
  \]
  其中 `h_mk` 表示测点对故障源的综合敏感度，`delta_mk` 表示传播相位差，`n_m(t)` 表示测点噪声。
- 设定有限候选测点，不做连续空间优化：
  `bearing_left`, `bearing_right`, `gearbox_left`, `gearbox_right`, `input_shaft`, `output_shaft`。
- 枚举所有 1、2、3 传感器组合，计算每个布局的：
  - 综合检测概率 `P_D`
  - 经验误报率 `P_FA`
  - 各故障源检测概率均衡性
  - 低 SNR 鲁棒性评分
- 融合规则采用加权 GLRT：
  \[
  T_{\text{fused}}(f)=\sum_{m\in S}w_mT_m(f),\quad w_m\propto 1/\hat\sigma_m^2
  \]
  噪声越小的测点权重越高。
- 最优布局目标函数：
  \[
  J(S)=\overline{P_D(S)}-\lambda P_{FA}(S)-\mu\operatorname{Var}_k(P_{D,k}(S))
  \]
  默认 `lambda=2.0`，`mu=0.5`，优先避免误报和单一故障源偏科。

## Implementation Changes

- 新建 `q4_sensor_layout/`：
  - `core.py`：传感器场景、信号生成、融合统计量、布局评分。
  - `experiments.py`：SNR 扫描、布局枚举、鲁棒性实验。
  - `outputs.py`：CSV、PNG、Markdown 报告输出。
  - `run_q4.py`：命令行入口。
