# 强噪声下微弱故障信号检测报告

强噪声背景下的微弱故障检测，不宜理解为“找一个最强的单算法”，更接近于设计一条稳健的级联系统。文献的共同结论是：先补偿传递路径并增强周期冲击，再做有针对性的频带选择与解调，最后用明确的统计指标完成判决，通常比直接在原始信号上做包络谱或峭度分析更稳健 \[Mcd12, Buz18, Jia15, Smi19\]。

对数学建模比赛而言，报告的重点应放在“怎样提高检出率并压低误报率”。从现有文献看，误报最常见的来源不是故障太弱，而是把与故障无关的随机冲击、齿轮干扰或其他循环成分误当成故障 \[Ant06, Smi19\]。因此，工程设计的关键是让每一层都围绕故障的**周期结构**而不是单纯的**脉冲强度**来工作 \[Ran01, Mcd12, Buz18\]。

## 文献中的主要技术路线

| 路线 | 主要作用 | 优点 | 主要风险 |
|:---|:---|:---|:---|
| 谱峭度与 Kurtogram \[Ant06\] | 自动找高冲击共振频带 | 实现简单，适合早期筛查 | 易被非目标冲击误导，强噪声下误报风险高 |
| 包络谱与循环谱 \[Ran01\] | 提取周期调制与故障特征频率 | 对周期故障更有物理解释，判读清楚 | 频带若选错，效果明显下降 |
| MCKD 解卷积 \[Mcd12\] | 增强周期冲击并补偿传递路径 | 对低信噪比和周期冲击很有效 | 依赖故障周期先验，对转速波动敏感 |
| 循环平稳盲解卷积 \[Buz18\] | 在复杂传递路径下恢复冲击序列 | 比单纯峭度型解卷积更稳健 | 计算更重，仍需要循环频率先验 |
| 针对性频带选择 \[Smi19\] | 在干扰下选对解调频带 | 比传统 kurtogram 更抗误选频带 | 需要已知或可估计故障特征频率 |
| 自适应滤波 + LMD \[Dar18\] | 压制强背景噪声并提取非平稳特征 | 对强随机噪声场景实用 | 物理解释性弱于循环平稳路线 |

## 对比赛建模最有用的结论

\[Ran01\] 的核心价值在于说明：包络分析本质上是循环平稳分析的简化投影，因此真正稳定的检出，依赖的是故障调制的周期结构，而不是单次冲击的幅值。这个结论很适合写进建模假设：故障信号可被视为被结构传递路径调制后的近周期冲击序列。

\[Ant06\] 建立了谱峭度与 kurtogram 这套经典频带选择框架，但这篇论文本身也揭示了它的边界：强噪声会压缩谱峭度，非故障冲击也会产生高峭度，从而造成误选频带。若比赛题目强调低误报率，则不宜把 kurtogram 当成最终判决层，更适合把它作为候选频带生成器。

\[Mcd12\] 和 \[Jia15\] 支持一个很实用的工程链路：先用 MCKD 利用故障周期增强微弱冲击，再做 kurtogram 或包络分析。这样做的原因不是“先去噪后分析”这么简单，而是先通过周期约束把故障冲击从随机噪声中拉出来，使后续的频带定位和解调不再完全依赖原始信号的脉冲性。

\[Buz18\] 进一步表明，若比赛场景存在复杂传递路径、齿轮干扰或转速变化，单纯最大峭度并不稳健，最大化循环平稳性的盲解卷积更接近故障形成机理。这类方法适合作为高阶方案：当基础链路效果不足时，再引入循环平稳盲解卷积替代 MCKD。

\[Smi19\] 对比赛最有价值的一点是：频带选择必须“针对已知故障频率”。文中比较表明，传统 kurtogram 和 protrugram 在无关冲击或无关循环干扰下容易误选频带，而面向已知故障频率的 targeted 方法更稳健。因此，如果题目给出转速、轴承几何参数或理论故障频率，模型中应显式利用这些先验，而不是做完全盲检。

\[Dar18\] 提供了另一条适合强随机噪声的路线：先用自适应滤波抑制背景噪声，再做分解和解调。若赛题数据更像“强噪声淹没弱特征”，而不是“明显周期冲击经过复杂传递路径”，这条路线可能比复杂解卷积更易实现。

## 推荐的工程检测系统

| 阶段 | 推荐做法 | 设计目的 |
|:---|:---|:---|
| 预处理 | 去均值、去趋势、必要时重采样或阶次跟踪 | 降低工况漂移和转速波动影响 \[Ran01, Buz18\] |
| 冲击增强 | 首选 MCKD；转速变化或路径复杂时考虑循环平稳盲解卷积 | 提升微弱周期冲击可见性 \[Mcd12, Buz18, Jia15\] |
| 频带选择 | 先验已知时采用 targeted band selection；未知时再用 kurtogram 做候选筛查 | 降低误选频带概率 \[Ant06, Smi19\] |
| 解调特征 | 包络谱、平方包络谱或循环谱积分 | 提取故障特征频率及倍频 \[Ran01\] |
| 判决 | 用故障频率峰值、谐波一致性、边带结构和背景比值联合判决 | 提高检出率并压低误报率 \[Ran01, Smi19\] |

## 可直接写进模型的目标与指标

若把系统输出记为检测统计量 $`T`$，则建模目标可写为：在满足误报率约束 $`P_{FA} \le \alpha`$ 的前提下，最大化检测概率 $`P_D`$。

``` math
\max P_D \quad \text{s.t.} \quad P_{FA} \le \alpha
```

与文献更一致的做法，不是只优化单个峰值，而是构造一个综合统计量。可考虑把以下信息加权融合：

- 故障特征频率及其倍频峰值幅值
- 谐波间一致性
- 预期边带是否出现
- 目标频带内信号能量与邻域背景能量之比
- 多时间窗下统计量的稳定性

这样做的依据是：\[Ran01\] 强调故障识别依赖周期结构，\[Smi19\] 强调频带选择必须避免无关冲击，\[Mcd12\] 和 \[Buz18\] 则说明前端增强应尽量围绕周期冲击而非孤立脉冲。

## 比赛中可采用的两级方案

| 方案 | 适用情况 | 组成 |
|:---|:---|:---|
| 基础方案 | 需要快速实现、可解释性强 | 预处理 + MCKD + targeted band selection + 包络谱判决 \[Mcd12, Jia15, Smi19\] |
| 提升方案 | 噪声更强、传递路径更复杂、转速不稳 | 阶次跟踪 + 循环平稳盲解卷积 + 循环谱/包络谱联合判决 \[Buz18, Ran01\] |

## 应避免的常见误区

- 仅凭最大峭度频带直接下故障结论，这在强冲击噪声下很容易误报 \[Ant06, Smi19\]。
- 只看单个故障频率峰值，不检查谐波和边带一致性，会降低判决稳健性 \[Ran01\]。
- 在未知转速波动的情况下直接使用固定周期的 MCKD，可能导致增强失败 \[Mcd12, Buz18\]。
- 把自适应滤波、解卷积和盲分离并联堆叠，而没有明确每一步的作用，会使模型复杂但不稳健。

## 结论

从工程实现角度看，强噪声下微弱故障检测的最优思路不是追求最复杂的单方法，而是构建一条围绕“周期冲击增强—稳健频带选择—解调判决”的检测链路。若比赛数据具有明确转速和故障特征频率，推荐以 MCKD 或循环平稳盲解卷积为前端增强、以 targeted band selection 和包络谱判决为主干 \[Mcd12, Buz18, Smi19\]。若数据更偏强随机噪声淹没型，则可把自适应滤波作为前端补充 \[Dar18\]。整体上，能否显式利用故障周期先验，是提高检出率并压低误报率的关键。
