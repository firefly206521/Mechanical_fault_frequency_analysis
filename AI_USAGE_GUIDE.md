## 一、正文标注要求

论文中使用 AI 生成内容时，在对应位置标注工具编号（编号与参考文献一致），例如：

```latex
% 代码段末尾
% [AI-1] 优化联合精修 cached backend 与并行调度，人工验证数值一致性

% 图表下方

```

**标注需精确到具体内容**（某段代码、某个图表、某处文字），禁止笼统标注"本文部分内容由 AI 生成"。

### 2.3 关键交互记录（代表性若干条）

**交互 1：Q1 FFT vs GLRT 等价性验证**
```
输入：为什么 FFT 和 GLRT 给出的频率误差逐位完全相等？是不是代码写错了？
输出：这大概率不是 bug。在单正弦 + 高斯白噪声模型下，GLRT 统计量
      T(f) = 2|FFT(f)|²/(Nσ²) 是周期图的正线性变换。二者 argmax 相等是数学必然。
      建议检查 Kay 1998 §7.6 确认。
采纳：确认数学等价后，论文相应段落从"模型驱动 vs 数据驱动"改为诚实表述。AI 辅助实现 MC 门限校准与样本长度统一。
```

**交互 2：Q3 误报率定义修正**
```
输入：纯噪声误报率约为千分量级，但只测了首步 GLRT 越阈率，issue 要求测完整 SIC+BIC 最终误报率
输出：需要新增独立实验 C，对 K=0 纯噪声跑完整 detect_multitone 流程。
      通过较大规模纯噪声 Monte Carlo 估计最终误报率，并用区间估计描述不确定性。
采纳：完整采纳，AI 辅助实现 run_q3_experiment_c.py，最终未观察到完整流程误报。
```

**交互 3：Q3 近频辨识极限曲面化**
```
输入：辨识极限不能只写单个固定数字，需要条件化（SNR × 振幅比 × 分离间隔）
输出：设计多因素网格和 Monte Carlo 试验，用完整 detect_multitone 而非 close_pair_resolver
      （后者在部分近频条件下产生非单调伪影，已废弃）。
采纳：完整采纳，AI 辅助实现 run_q3_experiment_d.py 和条件化经验界限表。
```

**交互 4：Q4 目标函数修正**
```
输入：V1 目标函数选出的布局 weakest-source 检测反而不如 V0，问题在哪？
输出：原目标混合了不同量纲的项（检测强度、冗余度、信息矩阵）。应改为融合 GLRT 检测代理，
      按逆噪声方差加权，冗余度项按检测强度尺度缩放。
采纳：完整采纳，修正后 weakest-source 指标有明显提升。
```

**交互 5：CPU 优化决策**
```
输入：Q3 实验 D 正式跑需要较长时间，要不要上 CUDA？
输出：先 profiling 找热点。单 trial 热点在联合频率精修（重复构建设计矩阵），用缓存替代。
      CUDA 不适合慢在 scipy 小数组 optimizer 的场景。
采纳：实现 cached backend，短测显示有明确 CPU 侧加速收益，未上 CUDA。
```

### 2.4 采纳与修改情况

| 内容类型 | AI 生成 | 直接采纳 | 经人工修改 | 未采纳 |
|---------|--------|---------|-----------|-------|
| 核心算法优化 | AI 辅助调试与性能调优 | 人工初版，AI 协助定位并修复多项问题 | — | AI 提出的若干误判 |
| 实验编排脚本 | AI 辅助参数网格与输出结构设计 | 大部分框架采纳 | 少量路径、参数默认值和口径由人工调整 | — |
| 论文文本草稿 | AI 辅助生成实验小节初稿 | 少量直接采纳 | 大部分由人工重写、判断和定稿 | — |
| 图表代码 | AI 优化配色与标注 | 大部分绘图代码采纳 | 中文字体、最终样式和取舍由人工调整 | — |
| 优化方案 | AI 评估多个方案 | 采纳 CPU 多进程与缓存优化方向 | — | 暂未采纳 Numba、CuPy 等方案 |

**代表性人工修改**：
1. Q2 相位 CI 边界：AI 初版直接对 wrapped phase 取分位数，人工修改为 `phase_in_circular_ci()` 圆环判断。
2. Q3 近频精修：AI 初版对每个候选频率独立 golden-section，人工改为 SIC 阶段 golden-section + 联合精修阶段 L-BFGS-B。
3. Q4 swap 搜索：AI 初版 `while improved` 无 max_iter，人工添加迭代上限和 warning。
4. 论文"Bootstrap"措辞：AI 初版沿用此术语，人工改为"局部渐近正态近似"（因实现是协方差抽样，非残差重采样）。

## 三、人工独立完成内容

AI 主要用于文献检索辅助、代码优化调试、图表润色、报告草稿和审查辅助。以下核心判断由参赛队完成：

| 工作内容 | 方式 |
|---------|------|
| 文献调研与算法选型 | 独立完成（见 `paper/强噪声下微弱故障信号检测报告.md`），AI 辅助文献检索 |
| 信号模型（正弦 + 高斯白噪声） | 独立建模 |
| GLRT 统计量推导与阈值理论 | 独立推导 |
| SIC 顺序检测框架 | 独立设计 |
| BIC 定阶准则 | 独立选择 |
| 传感器布局目标函数 | 独立设计 |
| 数值边界保护（log(0)、argmax([])、相位 CI） | 人工发现并修复 |
| 实验结果分析与论文结论 | 独立判断 |
| CPU 优化方案选择 | 独立决策 |

**说明：** 文献调研方向、物理建模、统计量推导、模型选择的数学论证和结论判断由参赛队完成；AI 仅用于资料整理、表达校核、代码实现辅助和结果复核建议。

代表性人工修正见第二章“采纳与修改情况”。
